## HIGH IMPACT

### H1 — No `prefers-reduced-motion` handling anywhere
- **Where:** `design-system.css:344, 365, 380, 395, 405, 535, 631, 668, 685, 838` — every keyframe animation (`stars-drift`, `stars-twinkle`, `nebula-drift`, `hermes-pulse`, `shimmer`, `spin-glow`, `float`, `tooltip-in`). Also `HermesAgentBar.tsx:224, 253, 449` (`animate-pulse`, `pulse` keyframe).
- **Problem:** Users with vestibular disorders get hit by 6 simultaneously looping bg animations and a pulsing pill. WCAG 2.3.3.
- **Fix:** Add a single global override at the bottom of `design-system.css`:
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      animation-iteration-count: 1 !important;
      transition-duration: 0.01ms !important;
      scroll-behavior: auto !important;
    }
    .stars-layer-1, .stars-layer-2, .stars-layer-3, .nebula { display: none; }
  }
  ```

### H2 — DataCenter.tsx bypasses the design system with raw Tailwind gray utilities on a dark background
- **Where:** `DataCenter.tsx:95, 96, 105, 113, 124, 140` — `text-gray-400`, `text-gray-500`, `text-gray-600`, `hover:text-gray-300`, `hover:bg-white/5`, `text-white/40`.
- **Problem:** `text-gray-500` on `#04050d` is ~3.4:1 contrast — fails WCAG AA for normal text. Also, this app doesn't use any `Card` / `SectionHeader` / `Button` / `EmptyState` primitives that every other app uses, so it visually breaks the polish contract.
- **Fix:** Replace with `style={{ color: 'var(--text3)' }}` / `var(--text2)` and use `<EmptyState>` instead of the bespoke div at line 138-141. Add a `Button variant="primary"` for "Analyze with Hermes" (line 126) and `<SectionHeader>` for the table header.

### H3 — Notes.tsx "new note" form has no Escape-to-close and no focus trap
- **Where:** `Notes.tsx:113-170` — the `showForm` overlay renders a full `<Card>` with `<Input>`/`<TextArea>`, but `press Escape` does nothing and focus is not auto-moved into the first field.
- **Fix:** In the new-note form block add `onKeyDown={e => e.key === 'Escape' && cancelCreate()}` on the wrapping div, plus `autoFocus` on the title `<Input>`.

## MEDIUM IMPACT

### M1 — HermesAgentBar chat input missing aria-label
- **Where:** `HermesAgentBar.tsx:462-483`. The only label is `placeholder="Ask Hermes…"` which screen readers ignore once typing starts.
- **Fix:** Add `aria-label="Ask Hermes"` on the `<input>` (line 462).

### M2 — Icon-only buttons rely on `title=` instead of `aria-label=`
- **Where:** `Taskbar.tsx:121 (start), 146 (running apps), 169 (AI), 178 (theme), 572 (volume)`, `HermesAgentBar.tsx:210 (minimized pill), 233 (trigger), 297 (minimize), 304 (close), 321 (tab buttons)`.
- **Problem:** `title` is inconsistent across screen readers. `aria-label` is the standard.
- **Fix:** Add explicit `aria-label` mirroring the `title` text on every icon-only `<button>`.

### M3 — Search overlay click-outside closes entire palette with no focus return
- **Where:** `Desktop.tsx:196-200`. The outer overlay's `onClick={() => setShowSearch(false)}` fires when users click any non-panel area. Focus is never restored to whichever element was focused before the palette opened.
- **Fix:** On open, `document.activeElement` should be saved to a ref; on close call `.focus()` back; bound to the `Escape` handler at line 81-84.

### M4 — Window close button has no global Escape-to-close
- **Where:** `Window.tsx` has no `keydown` listener. Users on desktop must drag to the X.
- **Fix (light-touch):** Add a `useEffect` in `Window.tsx` listening for `Escape` while the window is mounted+active, calling `closeWindow(win.id)`. Mirror with `Cmd/Ctrl+W`.

## LOW IMPACT (POLISH)

### L1 — Desktop icon fires *both* `onClick` and `onDoubleClick`
- **Where:** `Desktop.tsx:148-149` — `onDoubleClick` and `onClick` both call `handleAppOpen`. On mouse, single-click already launches the app, so dblclick is dead code; on touch it means a double-tap fires the handler twice.
- **Fix:** Drop `onDoubleClick` (already published by the existing `onClick`) or vice versa. Make one of them `noop` for clarity.

### L2 — ToastViewport z-index vs modal layering
- **Where:** `Toast.tsx` (imported at `Desktop.tsx:18`) — toasts default to stacking with root z-index; the search overlay is also a backdrop modal. The Taskbar's start menu uses `z-index: 9999` (Taskbar.tsx:69), ContextMenu uses `10000` (ContextMenu.tsx:116). If toasts ever need to appear *over* a modal they'd be buried. Not currently broken; flag for the next modal.
- **Fix:** Standardize: `--z-toast: 9998; --z-startmenu: 9999; --z-context: 10000; --z-modal: 10001` in design-system.css and reference them.

### L3 — `Card` interactive variant has no keyboard support
- **Where:** `components/ui/index.tsx:23-33` — `<Card onClick=…>` uses a `<div>` with `onClick`. No `role="button"`, no `tabIndex={0}`, no `onKeyDown` for Enter/Space.
- **Fix:** Add `role={clickable ? 'button' : undefined}`, `tabIndex={clickable ? 0 : undefined}`, and an `onKeyDown` handler that calls `onClick` on Enter/Space.

### L4 — Inconsistent transitions: Tailwind `transition-all duration-200` mixed with token `transition: all var(--dur-fast)`
- **Where:** `HermesAgentBar.tsx:235` (`transition-all duration-200`) vs surrounding code using `var(--dur-fast)`. Same in `Calculator.tsx` (search needed).
- **Fix:** Search-and-replace Tailwind transition utilities with the token equivalents, or define `--dur-base` Tailwind alias. Minor, but trims the bundle and keeps motion coherent.
